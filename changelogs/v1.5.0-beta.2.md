## Fixed

- **codex `/model` gpt-5.x visibility** (#1546, cherry-pick #1547,
  [@chenhg5](https://github.com/chenhg5))

  External user Shawn (WeChat) reported: "5.6的模型选不到，已经更新几次 cli 到 144 版本".

  **Root cause**: `agent/codex/codex.go` `openaiChatModels` static allowlist had only
  11 entries from the o3/o4/gpt-4o era (last updated 2026-03-02). When models are
  fetched via `GET /v1/models` from the provider API, all `gpt-5.x` variants were
  silently filtered out — users on codex-cli ≥ 0.143 could not pick GPT-5.6 in
  cc-connect's `/model` chooser.

  **Fix**: Replace static allowlist with prefix pattern matching:
  - Accept: `gpt-*`, `chatgpt-*`, `codex-*`, `o1-*`, `o3-*`, `o4-*`, `o5-*`
  - Reject non-chat modalities: embedding, whisper, tts, dall-e, audio-preview,
    realtime, transcribe, moderation, image, search-preview
  - Future-proof: gpt-6 / o5-* families auto-compatible

  **Blast radius**: Only affects codex agent `/model` chooser on the API-fetch
  fallback path. Users with explicit `model_catalog_json` or provider `models = [...]`
  config were unaffected. All legacy chat models retained.

  **QA evidence**: Owner E2E on Feishu test-feishu-codex with 24-model mock API:
  - BEFORE: 6 whitelist-intersection models only, all gpt-5.x filtered
  - AFTER: gpt-5.5, gpt-5.6-sol/terra/luna/fast visible; embedding/whisper/tts excluded
  - 45 new unit tests (`TestIsCodexChatModel`); CI green on main PR #1546
